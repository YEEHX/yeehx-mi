# 觅影素材库（miying）

觅影是导演玩椰的本地影视素材库（服务跑在 127.0.0.1:8788），已通过 MCP 注册 6 个工具。
铁律：**原始素材永远只读**——这些工具只会 搜索 / 出预览图 / 给路径 / 复制，没有任何能改动原素材的通道。

## 第一规则

**永远用 miying_* 工具，禁止自己写 Python/HTTP 脚本直连 8788 端口。**
工具内置了：服务自动拉起、AI 解析失败自动降级、超大文件转 1080p 代理、路径白名单。
裸写脚本 = 放弃全部保障，还容易写错。

## 工具清单

| 工具 | 干什么 | 关键点 |
|---|---|---|
| miying_search(query, limit=8) | 自然语言找素材 | 一切流程的起点，返回 文件名/id/位置/标签 |
| miying_preview(asset_ids) | 取代表帧 JPG | 逗号分隔 id，一次≤9 个；返回本地图片路径→**作为图片消息发送** |
| miying_get_original(asset_id) | 取原片用于发送 | 超 950MB 的视频自动转 1080p 代理（可能要几分钟，耐心等返回）；返回里「发送文件：<路径>」就是要发的文件 |
| miying_export(target_dir, asset_ids 或 query) | 批量复制原片到文件夹 | target_dir 必须绝对路径（mac 桌面=/Users/xxx/Desktop/...，Windows=C:\\Users\\xxx\\Desktop\\... 或 D:\\...）；只复制 |
| miying_status() | 版本/任务进度/待复核数 | 也可用来预热服务 |
| miying_reveal(asset_id) | 文件管理器（mac Finder / Win 资源管理器）中弹出选中原文件 | 人就在电脑前时用这个最快，不转码不复制 |

## 标准流程

1. 用户描述要什么 → `miying_search("雨中长江大桥航拍", limit=8)`
2. 把命中列表念给用户（文件名+标签），需要看效果 → `miying_preview("id1,id2")` 把返回的图片文件发出去
3. 用户确认要哪条 → 发原片用 `miying_get_original(id)`；批量拷走用 `miying_export("/Users/xxx/Desktop/选片", asset_ids="id1,id2")`（Windows 机器上路径形如 `D:\\选片`）
4. 随口问"觅影什么状态/打标完了吗" → `miying_status()`

## 边界与话术

- 卷离线：工具会返回素材的记录位置——原样转告用户"硬盘没挂载，回头插上盘再取"，不要瞎重试。
- 搜不到：换近义词（大桥→桥梁）、去掉限定词再搜一次；还没有就 `miying_status()` 看是否素材还在打标中。
- 导出目标只能在用户目录或各硬盘/盘符里，系统目录（Windows、Program Files 等）会被拒绝（403）——这是觅影的安全白名单，属预期。
- 不要替用户决定删除/移动任何东西：觅影没有这种工具，这是有意设计。
